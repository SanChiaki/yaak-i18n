# Yaak i18n Implementation

本文档说明了 Yaak 项目的国际化（i18n）实现。

## 概述

Yaak 现在支持多语言界面，目前包含：

- 英文（en）- 默认语言
- 简体中文（zh-CN）

## 技术栈

- **i18next**: 核心 i18n 框架
- **react-i18next**: React 集成
- **i18next-browser-languagedetector**: 浏览器语言检测

## 架构

### 1. 翻译文件组织

翻译文件按功能模块组织，位于 `apps/yaak-client/locales/`:

```
locales/
├── en/
│   ├── common.json      # 通用文本（按钮、标签等）
│   ├── settings.json    # 设置页面
│   ├── request.json     # 请求相关
│   ├── workspace.json   # 工作区相关
│   └── errors.json      # 错误消息
└── zh-CN/
    ├── common.json
    ├── settings.json
    ├── request.json
    ├── workspace.json
    └── errors.json
```

### 2. 语言检测策略

按优先级顺序：

1. **用户设置** - 如果用户在设置中明确选择了语言
2. **系统语言** - 通过 Tauri API 检测操作系统语言
3. **回退语言** - 默认英文（en）

系统语言映射：

- `zh-CN`, `zh-Hans`, `zh-SG` → 简体中文
- 其他 → 英文

### 3. 数据模型变更

在 Settings 模型中添加了 `language` 字段：

**Rust 模型** (`crates/yaak-models/src/models.rs`):

```rust
pub struct Settings {
    // ... 其他字段
    #[serde(default)]
    pub language: Option<String>,
}
```

**数据库迁移** (`crates/yaak-models/migrations/20260714000000_add-language-setting.sql`):

```sql
ALTER TABLE settings ADD COLUMN language TEXT;
```

### 4. 核心组件

#### i18n 配置 (`apps/yaak-client/i18n.ts`)

初始化 i18next，配置语言检测和翻译资源。

#### useLanguage Hook (`apps/yaak-client/hooks/useLanguage.ts`)

管理语言状态和切换：

- 检测系统语言
- 同步用户设置
- 提供语言切换函数

#### 语言选择器 (`components/Settings/SettingsInterface.tsx`)

在设置界面中提供语言选择下拉框。

## 使用方法

### 在组件中使用翻译

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation();

  return (
    <div>
      {/* 使用默认命名空间 (common) */}
      <button>{t("save")}</button>

      {/* 指定命名空间 */}
      <h1>{t("settings:title")}</h1>

      {/* 带变量插值 */}
      <p>{t("workspace:deleteConfirm", { name: "My Workspace" })}</p>
    </div>
  );
}
```

### 添加新的翻译

1. 在 `locales/en/[namespace].json` 中添加英文翻译
2. 在 `locales/zh-CN/[namespace].json` 中添加对应的中文翻译
3. 使用 `t('namespace:key')` 在组件中引用

### 翻译键命名规范

- 使用点号分隔层级：`settings.language.title`
- 命名空间与键之间用冒号：`settings:language.title`
- 使用描述性的键名，避免使用 `text1`, `label2` 等
- 插值变量使用 camelCase：`{{userName}}`, `{{itemCount}}`

### 技术术语处理

遵循混合策略：

- **保持英文**：REST, GraphQL, gRPC, WebSocket, OAuth, JWT, Cookie, JSON, XML
- **翻译中文**：保存、删除、创建、发送、工作区、环境、请求、响应
- **混合使用**："发送 HTTP 请求"、"创建 Workspace"

## TypeScript 类型安全

翻译键具有 TypeScript 类型检查支持（通过 `apps/yaak-client/i18n.d.ts`）：

```typescript
// ✓ 正确 - 键存在
t("common:save");

// ✗ 错误 - 键不存在（编译时报错）
t("common:savee");
```

## 日期和数字格式化

- **日期/时间**: 使用现有的 `date-fns` 库，可导入对应 locale
- **数字/货币**: 使用浏览器原生 `Intl.NumberFormat`
- i18next 专注于文本翻译

## 测试

运行项目并切换语言：

1. 启动应用：`npm run client:dev`
2. 打开设置 > 界面
3. 在"语言"下拉框中选择不同语言
4. 界面应立即切换语言

## 添加新语言

1. 在 `locales/` 下创建新的语言目录（如 `locales/ja/`）
2. 复制 `en/` 中的所有 JSON 文件并翻译
3. 在 `i18n.ts` 中导入并添加到 `resources`
4. 在 `SettingsInterface.tsx` 的语言选项中添加新选项

## 已知限制

- 第一阶段仅翻译核心功能界面
- Rust 后端消息暂时保持英文
- 插件描述和文档未翻译
